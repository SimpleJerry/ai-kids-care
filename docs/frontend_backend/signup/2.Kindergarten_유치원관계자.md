### 회원유형 유치원관계자가 선택된 상태에서 회원가입 버튼을 클릭하는 경우 
frontend,backend 매칭해서 작업

## ★ 특히 frontend 의 경우 지금 만들어져 있는 구조를 잘 분석해서, 
 components\auth\(signup)\KindergartenForm.tsx 파일을  생성.

## frontend
1) input box 대상 : 
이름, 로그인ID ,이메일, 전화번호, 비밀번호, 비밀번호 확인,유치원 찾기

2) 유치원 찾기 팝업 추가
input box는 1개 유치원사업자번호 또는 유치원명 입력 후 Enter 치면 유치원 찾기 팝업 호출됨.
해당 입력박스 옆에 유치원 찾기 버튼 추가해서 해당 버튼 클릭하면 유치원 찾기 팝업 호출.

조회 테이블 : kindergartens
유치원이름, 사업자번호 조회하고 kindergartens 테이블의 컬럼 활용해서 테이블 형태로 Display 함.


teachers 에 insert 할 kindergarten_id 컬럼의 값을 hidden 데이터로 화면에 가지고 있어야 함. 
팝업이 닫히면 화면에 kindergarten_id 셋업해 주어야 함.

3) input box 대상 
staff_no 직원(사번)
rrn_encrypted : 주민번호 뒷자리(암호화)
rrn_first6 : 주민등록번호 앞자리6

gender : 성별(MALE,FEMALE) -> 주민번호 뒷자리 읽어서 자동으로 성별 라디오 버튼에 선택되게 함.(자동셋팅)
emergency_contact_name : 비상 연락처 이름
emergency_contact_phone : 비상 연락처 전화번호

4) select box 
직급(level) - 원장-PRINCIPAL/부원장-VICE_PRINCIPAL/일반교사-TEACHER
Common_Code 테이블에서 teachers,level 조건으로 select box Display

5) 달력 
시작일, 종료일 유효성 체크 
start_date : 근무시작일
end_date : 근무종료일


## Backend

1.Database (postgresql)

insert table 4대상

1.1 users 테이블에 insert

user_id : 자동생성
login_id : 로그인 ID
email: 이메일
phone : 전화번호
password_hash : 비밀번호 -> jwt 를 활용하여 암호화 시켜서 저장할수 있게 함.
status : ACTIVE
last_login_at : 현재시간
created_at : 현재시간
updated_at : 현재시간

1.2 teachers 테이블 insert
teacher_id : 자동생성
kindergarten_id : 유치원찾기 통해서 나온 kindergarten_id hidden 값
user_id : users.user_id
staff_no : 직원(사번)
name : 이름
gender : 성별(MALE,FEMALE)
emergency_contact_name : 비상 연락처 이름
emergency_contact_phone : 비상 연락처 전화번호
rrn_encrypted : 주민번호 뒷자리(암호화화)
rrn_first6 : 주민등록번호 앞자리6
level : PRINCIPAL,VICE_PRINCIPAL,TEACHER 중 1개 값
start_date : 근무시작일
end_date : 근무종료일
status : 재직상태(ACTIVE)
created_at : 현재시간
updated_at : 현재시간


1.3 user_role_assignment insert
  role_assignment_id : increment
  user_id : users.user_id
  role user_role_enum : teachers.level 이 PRINCIPAL 이면 KINDERGARTEN_ADMIN 그 외는  TEACHER
  scope_type : KINDERGARTEN
  scope_id : kindergarten_id
  status status_enum : ACTIVE
  granted_at : 현재시간
  granted_by_user_id : users.user_id
  revoked_at : null

1.4 user_kindergarten_memberships insert
insert 조건 : 직급을 PRINCIPAL 유치원 원장 선택했을때
membership_id : increment
user_id : users.user_id
kindergarten_id: kindergarten_id
status : ACTIVE
joined_at : 현재시간
left_at : null
created_at :현재시간
updated_at :현재시간
